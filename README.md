# demonic-pacts-randomizer

A task randomizer for the OSRS [Demonic Pacts League](https://oldschool.runescape.wiki/w/Demonic_Pacts_League).

## What it does

- **Roll** one random task per difficulty tier (Easy/Medium/Hard/Elite/Master), filtered to your unlocked regions and excluding tasks you've already completed.
- **Pick one of the five** — that becomes your locked active task. You cannot roll or pick anything else until you mark it complete.
- **Sync** completed tasks via [WikiSync](https://oldschool.runescape.wiki/w/RuneScape:WikiSync) (RuneLite plugin) or import a JSON export from the [Tasks Tracker](https://github.com/osrs-reldo/tasks-tracker-plugin) plugin. Manual checkboxes always available as a fallback.
- **Categories tab** groups remaining tasks ("Slayer", "Bossing", "Skilling", "Quests", …) so you can scan upcoming work and pick a strategic candidate when rolling.

## Develop

```bash
npm install
npm run fetch:tasks   # pull latest from wiki into src/data/tasks.json
npm run dev           # http://localhost:5173
npm test
npm run build
```

## Data

`src/data/tasks.json` is generated by `scripts/fetch-tasks.mjs` from the wiki page's wikitext (parses `{{DPLTaskRow|...}}` templates). The file is committed so production builds work without live wiki access. A weekly GitHub Action refreshes it via PR.

## Deploy

`.github/workflows/deploy.yml` builds and deploys to GitHub Pages on push to `main`. Enable Pages in repo settings (source: GitHub Actions).

## CORS

WikiSync's read endpoint is intended for the wiki itself, not third parties. If a direct browser fetch is blocked, the app surfaces the error and the user can fall back to the Tasks Tracker plugin import — or deploy the small Cloudflare Worker proxy in [docs/cors-worker.md](docs/cors-worker.md).
